Нужно реализовать виджеты сервера и клиента, которые взаимодействуют по UDP (QUdpSocket).

Сервер должен передавать значение высоты с частотой 25 Гц клиенту. Значение высоты должно меняться ползунком 
на виджете сервера.

Клиент должен принимать значение высоты от сервера и обновлять свою картинку. Также клиент должен оповещать сервер о своей работе с частой 2 Гц.

Сервер и клиент - это отдельные Qt виджеты (названия классов виджетов - UdpServer, UdpClient).

Виджет клиента состоит из двух надписей (наличие связи с сервером и текущей высоты) и одной картинки.

Должна быть возможность изменить размер окна виджета клиента, при этом картинка индикатора высоты 
должна занимать все свободное пространство виджета.

Для отрисовки картинки в виджете клиента нужно реализовать отдельный виджет HeightIndicatorWidget.

Всю отрисовку картинки индикатора высоты нужно сделать с помощью QPainter.

Размер шрифта для вывода текста нужно подбирать автоматически в зависимости от размера виджета.

Интерфейс данного класса должен быть таким:

class HeightIndicatorWidget : public QWidget

{

Q_OBJECT

public:

HeightIndicatorImage(QObject* parent = nullptr);

void setHeight(int height);

protected:

void paintEvent(QPaintEvent*) override;

private:

int m_height = 0;

};

Отрисовку стрелок высоты можно сделать прямоугольниками.

Клиент должен принимать высоту по UDP (QUdpSocket) от сервера, порт приема задавать через файл settings.ini.

Формат сообщения представлен структурой Message1:

#pragma pack(push, 1)

struct Message1

{

quint16 header = 0xABCD;

quint16 height;

};

#pragma pack(pop)

Клиент должен сигнализировать серверу о своей работе.

Для этого клиент должен отправлять в сервер сообщение Message2 с частотой 2 Гц:

#pragma pack(push, 1)

struct Message2

{

quint16 header = 0x1234;

};

#pragma pack(pop)

Адрес и порт сервера, на который нужно посылать данное сообщение, нужно задавать через файл settings.ini (значения по умолчанию: адрес=127.0.0.1, порт=8001).

На виджете клиента нужно выводить признак того, что связь с сервером есть.

Если от сервера в течение последних 2 секунд не получено ни одного сообщения, то считается что связи с сервером нет. В обратном случае связь есть.

Сервер должен отправлять в клиент по UDP (QUdpSocket) значение текущей высоты, которая задается ползунком на виджете сервера, с частотой 25 Гц.

Для этого нужно использовать структуру Message1.

Адрес и порт клиента, на который нужно посылать данное сообщение, нужно задавать через файл settings.ini (значения по умолчанию: адрес=127.0.0.1, порт=8002).

Сервер должен принимать от клиента сообщение Message2.

По наличию или отсутствию данного сообщения сервер должен определять, есть ли связь с клиентом.

Если от клиента в течение последних 2 секунд не получено ни одного сообщения, то считается что связи с клиентом нет. В обратном случае связь есть.

Данную информацию нужно выводить на виджете сервера.
